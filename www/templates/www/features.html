About

PipeTaxon exposes the ncbi taxonomy database as a REST API. It's intended to be consumed by bioinformatic pipelines or dataviz applications. It allows you to run your own copy of the ncbi database and give your project data stability, minimizing the risks of changes in taxonomy data hosted in ncbi and given you the control over when the updates should be applied. It also supports custom lineage creation with predefined ranks, reducing the complexity in your pipeline to handle/remove undesired rank data.

Features

  • Retrieve taxonomy information by taxid
  • Find taxonomy by name string
  • Filter taxonomies by rank, division or parent
  • Retrieve entire lineage of taxonomy by taxid
  • Simple LCA endpoint
  • Easy to deploy
  • Supports Postgres, Mysql, SQLITE
  • Allow build database using selected classification rank

Rate limit

Anonymous requests to this API have a very low rate limit of 100 req/hour and should be used only for testing and validation, if you want to consume the API in production you can overcome this low rate limit by getting a token or running a self hosted version of PipeTaxon.

Support

To get support you can open an issue on github or send an e-mail to pipetaxon@voorloop.com

Contributing

We are always open to pull requests, but keep in mind that (for now) pipetaxon is supposed to be a generic taxonomy api for NCBI database, so if your pull request is rejected don't take it personal, maybe your changes sounded too much specific and probably could live happily in a fork.

Donating

If you liked pipetaxon and want it to thrive, feel free to donate. All money donated is used either for feature improvements or server capability.
